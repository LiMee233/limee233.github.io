删除「SIM n 支持高清」通知

2021/05/17
阅读时长: 2 分钟

出现原因与现象

部分原生系统在编译时，未考虑到会出现高清通知的问题 或者认为这不是问题 ，就没有在源码部分进行修改（比如 这里 ）。因此在手机插入 SIM 卡，且启用 Volte 功能（此功能默认开启）后，会在通知栏显示一条或多条（如果你插了很多 SIM 卡）「 SIM n 支持高清 」的通知，这条通知无法被用户彻底隐藏。

解决方案

如果作为用户，已经无法对系统编译前的 dt 进行修改了，所以只能修改编译好的应用程序，这里从通知的产生者—— ims.apk 入手。

警告
进行任何操作前，请注意对重要数据进行备份！

提取产生通知的应用

使用可获取 Root 权限的文件管理器，将 /system/product/priv-app/ims/ims.apk 复制到用户目录内（如 /data/media/0 ）。

修改应用

  1. 使用「 MT 管理器」打开复制的 ims.apk ，依次进入：查看 → resources.arsc → Arsc编辑器 → org.codeaurora.ims → string → string-zh-rCN. 在其中找到类似于「 SIM %s 支持高清 」的项目，长按并复制 ID.
  2. 返回到「查看」页面，依次进入：classes.dex → Dex 编辑器 ++ → 搜索 → 发起新搜索；在查找内容里粘贴刚才复制的「 ID 」并进行搜索，随后点击 const v5, 0xID

  1. 进入代码编辑界面后从所在行向上翻找，直到发现 .method private showHDIcon(Z)V 为止。
  2. 向下逐行查阅代码，去寻找如下结构，并将遇到的第一个如下结构删除。
if-ne xxxx
return-void

注意
删除的时候需要将整行删除，不要只删掉文字，也不要删多了。

修改完成后保存所做的一切修改，直到获取修改完的应用程序安装包

注意
修改完的应用程序不需要重新签名。

替换还原

方法一（推荐）

直接将修改完的应用移回 /system/product/priv-app/ims/ims.apk ，需要注意保持文件权限不变。

这样做可以让系统尽量保持一个完整的状态而不分散，个人感觉有利于系统稳定性。

方法二

制作Magisk模块进行替换（此文不阐述，可以拆一个 Magisk 替换应用的模块照抄）。
